# Contributing

感谢你为这个仓库补充学习资料。

这个仓库当前的主要目标不是继续还原更多源码，而是围绕现有内容，持续建设一套更适合开源读者的学习型资料库。

## 当前贡献重点

欢迎优先补充这几类内容：

1. 学习文档  
   例如新的专题导读、模块拆解、术语表、阅读指南、FAQ

2. 架构材料  
   例如 `.drawio` 架构图、流程图、模块关系图、序列图

3. 学习辅助资料  
   例如目录索引、源码阅读顺序建议、二开入口说明、模块地图

4. 文档修订  
   例如修正错误表述、补充缺失背景、统一术语、优化结构

## 不建议提交的内容

以下内容不属于当前仓库的优先方向：

- 与学习目标无关的随手脚本
- 本地 IDE 配置文件
- 含个人环境信息的截图、日志、路径
- 不能公开分发的账号、令牌、私有地址、私有仓库引用
- 把仓库包装成“官方源码”或“官方实现说明”的宣传性内容

## 文档写作原则

请尽量遵守下面这些规则：

### 1. 面向学习，而不是面向炫技

文档的目标是帮助读者建立认知，不是堆术语或复制源码。

优先写清楚：

- 这个模块是做什么的
- 它和上下游模块的边界是什么
- 为什么阅读顺序要这样安排
- 对应源码入口在哪里

### 2. 使用相对路径

仓库会开源，文档里不要出现本机绝对路径。

正确示例：

- `docs/day-01-learning-plan.md`
- `restored-src/src/query.ts`
- `../restored-src/src/`

避免使用：

- `D:/workspace/...`
- `C:/Users/...`
- 任何带个人目录结构的绝对路径

### 3. 不要包含敏感或隐私信息

请确保提交内容不包含：

- 本机用户名
- 本地磁盘路径
- 私有仓库地址
- API key、token、cookie、session
- 未公开的内部沟通信息
- 可识别个人身份的信息

如果资料来自你自己的阅读过程，请在提交前自行检查一遍。

### 4. 优先总结，不要大段复制源码

这是学习仓库，不是源码镜像注释仓库。

建议：

- 用自己的话总结
- 少量引用文件名和函数名
- 避免大段粘贴源码
- 避免把文档写成逐行翻译

### 5. 明确区分“事实”和“理解”

推荐写法：

- 事实：某文件负责什么、某函数在什么链路中
- 理解：为什么这样设计、建议怎么阅读

如果某处是你的推断，最好明确写成“可以理解为”或“建议这样看”。

## 推荐目录约定

如果继续补学习资料，建议沿用下面的目录组织：

```text
docs/
  README.md
  day-01-learning-plan.md
  ...
  day-14-learning-plan.md
  claude-code-architecture.drawio
```

如果后续资料变多，可以继续扩展为：

```text
docs/
  guides/          专题导读
  architecture/    架构图与说明
  glossary/        术语表
  modules/         模块级学习笔记
  faq/             常见问题
```

## 新文档建议模板

如果你要新增一篇学习文档，建议至少包含下面几个部分：

1. 这篇文档要解决什么问题
2. 推荐先读哪些源码文件
3. 核心结论
4. 阅读顺序
5. 术语解释
6. 常见误区
7. 继续深入应该看哪里

## 架构图贡献建议

如果你提交 `.drawio` 架构图，请尽量满足：

- 图中节点名称与仓库真实文件或模块一致
- 优先展示主链，而不是把所有边角功能都塞进去
- 主图和说明文字分开，避免一张图过载
- 图中不要出现本机路径或个人信息
- 如果修改现有图，请同时说明这次调整的意图

## 修改 README 的建议

`README.md` 目前主要承担 3 个作用：

1. 说明仓库性质
2. 给新读者提供学习入口
3. 把源码阅读导向 `restored-src/src/`

如果你修改 README，请尽量保持这 3 个目标不变，不要把首页写成冗长说明文，也不要把仓库定位改成“官方源码仓库”。

## 提交前自查

提交前建议至少检查：

1. 文档链接是否为相对路径
2. 是否出现绝对本机路径
3. 是否包含敏感或隐私信息
4. 术语是否前后一致
5. 文档是否真的帮助学习，而不是只增加字数

可以用下面这类命令快速自查：

```bash
rg "D:/|C:/Users|/Users/" docs README.md CONTRIBUTING.md
rg "token|apikey|cookie|session" docs README.md CONTRIBUTING.md
```

## 提交信息建议

建议 commit message 尽量直接：

- `docs: add module learning guide for query pipeline`
- `docs: refine readme learning navigation`
- `docs: add architecture diagram for tool execution`
- `docs: normalize links to relative paths`

## 如果你不确定写什么

如果你暂时不知道该补什么，优先从下面 3 个方向选一个：

1. 补一个专题导读  
   例如 `query`、`tools`、`session restore`、`MCP`

2. 补一个术语表  
   例如 command、tool、skill、plugin、MCP、hook、transcript

3. 补一个“二开入口说明”  
   例如“如果我要改权限，应该先看哪些文件”

## 最后说明

这个仓库欢迎继续补学习资料，但请始终记住它的定位：

它是一个围绕还原源码建立的学习型仓库，不是官方开发仓库，也不是用于分发敏感信息的地方。
