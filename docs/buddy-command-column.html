/buddy 指令不是一条普通命令

这页专门拆解本项目里 /buddy 的真实定位：它从 slash command 入口开始， 最终落到 companion 系统、消息上下文注入、REPL 底部 sprite、以及 turn 结束后的观察者 reaction。 你可以把它理解成“用一条命令点亮一套 companion 子系统”。

注册位置

src/commands.ts 在 feature('BUDDY') 打开时装载内建命令。

运行阶段

命令入口主要落在 slash command 分流，但副作用会继续影响消息构造与 UI 渲染。

持久化内容

真正写入 config 的是 soul：名字、性格、hatchedAt；骨架外观按 userId 重算。

系统角色

既是 UI companion，也是会影响模型行为边界的上下文附件来源。
